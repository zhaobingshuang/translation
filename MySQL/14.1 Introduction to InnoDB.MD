InnoDB 是一种兼顾高可靠性和高性能的通用存储引擎。在 MySQL 5.7中，InnoDB 是 MySQL 的默认存储引擎。除非您配置了其他默认存储引擎，否则执行不带 ENGINE 子句的 [`CREATE TABLE`](https://dev.mysql.com/doc/refman/5.7/en/create-table.html) 语句将创建一个 InnoDB 表。

### Key Advantages of InnoDB （主要优点）

- 其 DML 操作遵循 ACID 模型，并具有事务的提交、回滚和以保护用户数据为目的的崩溃恢复功能。See [Section 14.2, “InnoDB and the ACID Model”](https://dev.mysql.com/doc/refman/5.7/en/mysql-acid.html)。

- 行级锁和 Oracle 风格的一致性读可提高多用户的并发性和性能。 See [Section 14.7, “InnoDB Locking and Transaction Model”](https://dev.mysql.com/doc/refman/5.7/en/innodb-locking-transaction-model.html)。

-  InnoDB 表将数据安排在磁盘上，以优化基于主键的查询。每个 InnoDB 表都有一个称为聚集索引的主键索引，该索引以最小化主键查找的I / O为目的来组织数据。See [Section 14.6.2.1, “Clustered and Secondary Indexes”](https://dev.mysql.com/doc/refman/5.7/en/innodb-index-types.html)。 

- 为了保持数据完整性，InnoDB 支持 FOREIGN KEY 约束。使用外键时，会检查插入、更新和删除，以确保它们不会导致相关表之间的不一致。See [Section 14.6.2.1, “Clustered and Secondary Indexes”](https://dev.mysql.com/doc/refman/5.7/en/innodb-index-types.html)。

  

**Table 14.1 InnoDB Storage Engine Features**

| Feature                                                      | Support                                                      |
| :----------------------------------------------------------- | :----------------------------------------------------------- |
| **B-tree indexes**                                           | Yes                                                          |
| **Backup/point-in-time recovery** (Implemented in the server, rather than in the storage engine.) | Yes                                                          |
| **Cluster database support**                                 | No                                                           |
| **Clustered indexes**                                        | Yes                                                          |
| **Compressed data**                                          | Yes                                                          |
| **Data caches**                                              | Yes                                                          |
| **Encrypted data**                                           | Yes (Implemented in the server via encryption functions; In MySQL 5.7 and later, data-at-rest encryption is supported.) |
| **Foreign key support**                                      | Yes                                                          |
| **Full-text search indexes**                                 | Yes (Support for FULLTEXT indexes is available in MySQL 5.6 and later.) |
| **Geospatial data type support**                             | Yes                                                          |
| **Geospatial indexing support**                              | Yes (Support for geospatial indexing is available in MySQL 5.7 and later.) |
| **Hash indexes**                                             | No (InnoDB utilizes hash indexes internally for its Adaptive Hash Index feature.) |
| **Index caches**                                             | Yes                                                          |
| **Locking granularity**                                      | Row                                                          |
| **MVCC**                                                     | Yes                                                          |
| **Replication support** (Implemented in the server, rather than in the storage engine.) | Yes                                                          |
| **Storage limits**                                           | 64TB                                                         |
| **T-tree indexes**                                           | No                                                           |
| **Transactions**                                             | Yes                                                          |
| **Update statistics for data dictionary**                    | Yes                                                          |
